## 🚀 Introducción a Docker en Ubuntu

Docker se ha convertido en una herramienta esencial para desarrolladores y equipos DevOps. Permite crear, implementar y ejecutar aplicaciones en contenedores, asegurando consistencia y portabilidad. Si estás trabajando en un entorno Ubuntu, instalar Docker es un paso fundamental para aprovechar estas ventajas.

## 🤔 ¿Por qué Docker en Ubuntu?

Ubuntu es una de las distribuciones de Linux más populares, conocida por su estabilidad y amplio soporte de comunidad. Docker, por otro lado, es la plataforma líder para la gestión de contenedores. Juntos, proporcionan un entorno robusto y eficiente para desarrollar y desplegar aplicaciones.

### Beneficios de usar Docker en Ubuntu

- **Consistencia**: Asegura que las aplicaciones se ejecuten de la misma manera en cualquier entorno.
- **Aislamiento**: Los contenedores Docker funcionan de manera independiente, evitando conflictos de dependencias.
- **Portabilidad**: Fácilmente trasladable entre diferentes sistemas que soportan Docker.
- **Eficiencia**: Los contenedores son ligeros y rápidos comparados con las máquinas virtuales tradicionales.

## ⚡ Preparativos antes de la instalación

Antes de proceder con la instalación de Docker, hay algunos pasos previos que debes completar para asegurar una instalación sin problemas.

### Actualización del sistema

Primero, asegúrate de que tu sistema Ubuntu esté actualizado. Esto garantiza que todas las dependencias necesarias estén en su versión más reciente.

```bash
sudo apt update
sudo apt upgrade
```

### Eliminación de versiones antiguas de Docker

Si tienes versiones anteriores de Docker instaladas, es recomendable eliminarlas para evitar conflictos.

```bash
sudo apt remove docker docker-engine docker.io containerd runc
```

## 🛠️ Instalación de Docker en Ubuntu

Ahora que tu sistema está listo, pasemos a la instalación de Docker.

### Paso 1: Instalación de dependencias

Docker requiere ciertos paquetes para funcionar correctamente. Instálalos con el siguiente comando:

```bash
sudo apt install apt-transport-https ca-certificates curl software-properties-common
```

### Paso 2: Añadir la clave GPG de Docker

Para asegurarnos de descargar versiones seguras de Docker, añadimos su clave GPG oficial:

```bash
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
```

### Paso 3: Añadir el repositorio de Docker

Añadimos el repositorio oficial de Docker a las fuentes de APT:

```bash
sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"
```

### Paso 4: Instalación de Docker

Actualizamos el índice de paquetes y procedemos a instalar Docker:

```bash
sudo apt update
sudo apt install docker-ce
```

### Paso 5: Verificación de la instalación

Para verificar que Docker se instaló correctamente, ejecuta:

```bash
sudo systemctl status docker
```

## 💡 Configuración post-instalación

Después de instalar Docker, hay algunos pasos adicionales que puedes considerar para optimizar su uso.

### Ejecutar Docker como usuario no root

Por defecto, Docker requiere permisos de superusuario. Sin embargo, puedes configurar tu usuario para ejecutar Docker sin `sudo`:

```bash
sudo usermod -aG docker ${USER}
```

Después de ejecutar este comando, cierra sesión y vuelve a iniciarla para aplicar los cambios.

### Prueba de Docker

Para confirmar que Docker está funcionando correctamente, ejecuta el siguiente comando para descargar y ejecutar una imagen de prueba:

```bash
docker run hello-world
```

Si todo está bien, verás un mensaje de bienvenida de Docker.

## 🛠️ Solución de problemas comunes

### Problemas de permisos

Si recibes un error relacionado con permisos, asegúrate de haber agregado tu usuario al grupo Docker y reiniciado la sesión.

### Errores de red

Algunas veces, Docker puede tener problemas para acceder a internet. Verifica tu conexión y asegúrate de que no haya restricciones de firewall.
